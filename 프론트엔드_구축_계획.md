## 구축할 프론트엔드 기능

### 1. 시간대별 방문자 / 페이지뷰 추이 차트

**위치**: `src/views/admin/zestAnalytics/components/HourlyTrafficChart.jsx`

**UI 구성**
- 탭 또는 토글: 방문자 / 페이지뷰 전환
- 0~23시 막대 차트 (ApexCharts BarChart)
- x축: 0시~23시, y축: 방문자 수 or 페이지뷰 수
- DateRangeContext 연동 (날짜 범위 변경 시 자동 갱신)
- 피크 시간대 강조 (가장 높은 막대 색상 구분)

**데이터 흐름**
```
DateRangeContext → getHourlyVisitors() / getHourlyPageViews()
→ 24개 항목 배열 → BarChart
```

---

### 2. 페이지별 스크롤 도달률 테이블

**위치**: `src/views/admin/zestAnalytics/components/PageScrollStats.jsx`

**UI 구성**
- 테이블 (페이지 URL / 총 세션 / 평균 도달 깊이 / 25% / 50% / 75% / 100% 도달률)
- URL이 길 경우 말줄임 처리 + hover 시 full URL 표시
- 도달률 열: 숫자(%) + 프로그레스바 (색상 gradient: 낮으면 빨강 → 높으면 초록)
- 총 세션 기준 내림차순 정렬
- DateRangeContext 연동

**데이터 흐름**
```
DateRangeContext → getPageScrollStats()
→ page_url / total_sessions / reach_25_pct ~ reach_100_pct
→ 테이블 렌더링
```

---

### 3. 유입 경로별 성과 테이블

**위치**: `src/views/admin/zestAnalytics/components/ChannelPerformance.jsx`

**UI 구성**
- 테이블 (채널 / 페이지뷰 / 고유 방문자 / 평균 체류시간 / 평균 스크롤 도달 깊이 / 25~100% 도달률)
- 채널명에 아이콘 뱃지 (google_ads → 구글 색상, naver → 초록, direct → 회색 등)
- 체류시간: 초 → "X분 Y초" 포맷으로 변환
- 페이지뷰 기준 내림차순 정렬
- 성과 좋은 채널 하이라이트 (예: 스크롤 도달률 1위 채널에 왕관 아이콘)
- DateRangeContext 연동

**데이터 흐름**
```
DateRangeContext → getChannelPerformance()
→ channel / pageview_count / avg_time_on_page / reach_X_pct 등
→ 테이블 렌더링
```

### 4. ZestAnalytics 메인 페이지 탭 추가

**위치**: `src/views/admin/zestAnalytics/ZestAnalytics.jsx`

현재 탭 구조에 **트래픽 분석** 탭 추가:

```
[이벤트 통계] [어트리뷰션 분석] [캠페인 성과] [추적 코드 관리]  ← 기존
[트래픽 분석]  ← 신규 추가
```

**트래픽 분석 탭 내부 레이아웃**

└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│  유입 경로별 성과 테이블                 │  (3)
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│  페이지별 스크롤 도달률 테이블           │  (2)

## 공통 작업

### DateRange 연동
- 기존 `DateRangeContext`의 `startDate` / `endDate` 그대로 사용
- 날짜 변경 시 각 컴포넌트 자동 리패치

### 로딩/에러 처리
- 데이터 로딩 중: Chakra UI `Skeleton` 컴포넌트
- 에러 시: 에러 메시지 카드

### 광고주 필터
- 기존 `advertiserId` / `availableAdvertiserIds` 패턴 그대로 사용
