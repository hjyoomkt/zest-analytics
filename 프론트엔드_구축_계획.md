## ✅ 완료된 프론트엔드 기능

### 메인 대시보드 `/admin/default` (2026-04-05 구현 완료)

**위치**: `src/views/admin/default/index.jsx` + `src/views/admin/default/components/`

- 상단 날짜 선택 UI: `DateRangePicker.js` (react-calendar, 프리셋 드롭다운, 비교 모드)
- 6개 KPI 카드: 방문자수, 방문자당 페이지뷰, 평균 체류시간, 평균 스크롤 깊이, 신규방문, 재방문
- 일별 방문자 & 페이지뷰 이중 라인 차트: `VisitorTrendChart.jsx`
- 이탈률/새로고침률/뒤로가기율 MiniStatistics: `BehaviorRates.jsx`
- 기기별 도넛 차트: `DeviceStatsChart.jsx`
- 신규/재방문 도넛 차트: `VisitorTypeChart.jsx`
- 페이지별 방문 순위 (프로그레스바): `TopPages.jsx`
- 자주 하는 행동 Top10 (이벤트 배지): `TopActions.jsx`
- 유입경로 Top5 도넛 차트 + 목록: `TopReferrers.jsx`
- 브랜드 선택 → 전체 데이터 갱신 (AuthContext 연동)
- OS/브라우저 통계: `OsBrowserStats.jsx` (운영체제 + 브라우저 각각 프로그레스바, 이벤트 수 + 사용자 수 동시 표시)
- 탭 전환 시 데이터 리패치 방지: `useStableFetch.js` 훅

## 구축할 프론트엔드 기능

### 1. 시간대별 방문자 / 페이지뷰 추이 차트

**위치**: `src/views/admin/zestAnalytics/components/HourlyTrafficChart.jsx`

**UI 구성**
- 탭 또는 토글: 방문자 / 페이지뷰 전환
- 0~23시 막대 차트 (ApexCharts BarChart)
- x축: 0시~23시, y축: 방문자 수 or 페이지뷰 수
- DateRangeContext 연동 (날짜 범위 변경 시 자동 갱신)
- 피크 시간대 강조 (가장 높은 막대 색상 구분)

**데이터 흐름**
```
DateRangeContext → getHourlyVisitors() / getHourlyPageViews()
→ 24개 항목 배열 → BarChart
```

---

### 2. 페이지별 스크롤 도달률 테이블

**위치**: `src/views/admin/zestAnalytics/components/PageScrollStats.jsx`

**UI 구성**
- 테이블 (페이지 URL / 총 세션 / 평균 도달 깊이 / 25% / 50% / 75% / 100% 도달률)
- URL이 길 경우 말줄임 처리 + hover 시 full URL 표시
- 도달률 열: 숫자(%) + 프로그레스바 (색상 gradient: 낮으면 빨강 → 높으면 초록)
- 총 세션 기준 내림차순 정렬
- DateRangeContext 연동

**데이터 흐름**
```
DateRangeContext → getPageScrollStats()
→ page_url / total_sessions / reach_25_pct ~ reach_100_pct
→ 테이블 렌더링
```
