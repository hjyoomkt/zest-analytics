## 1. 요구사항: 채널별 히트맵에서 직접유입 및 채널별 선택 필터

**요청 내용**:
- `/admin/heatmap` → 채널별 히트맵 탭에서 `[직접유입(전체)]`, `[네이버]`, `[구글]` 등 채널을 선택하여 히트맵을 볼 수 있게 해달라

**현재 상태 (미완료)**:
- 채널별 히트맵은 현재 `page_url` 문자열에서 UTM 파라미터를 파싱하는 방식으로 채널 필터 칩을 표시 중
- DB의 `channel`, `utm_source`, `utm_medium`, `utm_campaign` 컬럼을 활용하지 않음

**근본 원인 발견 및 수정 완료**:
- SDK가 `pageview` / `session_end` 이벤트에 UTM 필드를 전송하지 않았음 → 수정 완료
- Edge Function이 전환 이벤트에만 UTM 저장 → 수정 완료
- `_detectChannel()`이 알 수 없는 utm_source를 그대로 `channel` 컬럼에 저장 → 수정 완료

**⚠️ 해결 못함**:
- 채널별 히트맵 자체의 DB 채널 필터 전환은 미완료
- SDK/Edge Function 버그 수정 후 신규 데이터가 충분히 쌓인 뒤 아래 작업 필요:

| 작업 | 파일 | 내용 |

| SQL 신규 | Supabase SQL Editor | `get_heatmap_channels` 함수 — `za_events.channel` 기준 목록 반환 |
| SQL 수정 | Supabase SQL Editor | `get_heatmap_page_list` — `p_channel` 파라미터 추가 |

## 2. 버그: SDK가 UTM 값을 DB 컬럼에 저장하지 않음

**증상**:
- UTM 파라미터(`?utm_source=ogb&utm_medium=cpc&utm_campaign=test`)를 달고 방문해도 `za_events` 테이블의 `utm_source`, `utm_medium`, `utm_campaign` 컬럼이 `null`
- `channel` 컬럼에 `utm_source` 원시값(`ogb`)이 그대로 저장됨

**원인**:
1. `za-sdk.js` — `trackPageView()`와 `_sendSessionEnd()`가 UTM 필드를 페이로드에 포함하지 않음
2. `za-collect-event.ts` — UTM 컬럼 저장이 `isConversion` 조건으로 막혀 있어 pageview/session_end는 항상 `null`
3. `_detectChannel()` — 알 수 없는 utm_source를 `return source`로 채널에 그대로 반환

**✅ 완전 해결 (2026-04-17)**:

**진짜 원인 발견**: CDN 캐시가 아니라 빌드 구조 문제였음
- 빌드 명령: `node src/views/admin/zestAnalytics/sdk/build.js && react-scripts build`
- `build.js`가 `src/.../sdk/index.js` → `public/sdk/za-sdk.js`로 덮어씀
- `4ac09a6` 커밋에서 `public/sdk/za-sdk.js`만 수정하고 `index.js`는 수정하지 않아 Vercel 빌드 시마다 구버전으로 복원됨

**수정 완료** (`a5f4d35` 커밋):
- `src/views/admin/zestAnalytics/sdk/index.js`: UTM 필드 추가, `_getStoredUtmParams()` 추가, `_detectChannel()` 버그 수정
- `public/sdk/za-sdk.js`: `build.js` 실행으로 동기화

**추가 개선** (`1ee0e9e` 커밋):
- Click ID 기반 채널 감지 추가 (gclid/fbclid/ttclid/twclid/n_media/NaPm/tb_clickid/cto_pld)
- meta → facebook 매핑 추가
- taboola, criteo, appier 채널 추가

**DB 확인 결과**: pageview/session_end 이벤트에 `utm_source`, `utm_medium`, `utm_campaign` 정상 저장 확인

## 3. 버그: Vercel CDN 캐시로 SDK 업데이트 미반영

**증상**:
- `za-sdk.js` 수정 후 git push + Vercel 재배포해도 `analytics.zestdot.com/sdk/za-sdk.js`가 구버전 반환
- `curl -I` 확인 결과 `X-Vercel-Cache: HIT`, `Age: 321`

**원인**:
- `public/` 폴더 정적 파일은 Vercel CDN에 캐싱되며, 재배포 시 즉시 무효화되지 않을 수 있음
- 기존 `vercel.json` 없어서 기본 캐시 정책 적용

**코드 수정 완료, 미반영** (`f3ca8a0` 커밋):
- `vercel.json` 신규 생성: `/sdk/*.js` 경로에 `Cache-Control: no-cache, no-store, must-revalidate` 설정

**✅ 완전 해결 (2026-04-17)**:
- 진짜 원인: CDN 캐시가 아니라 `index.js` 미수정이었음 (위 버그 2 참조)
- `vercel.json`의 `Cache-Control` 헤더는 정상 적용 중 (브라우저 캐시 방지)
- `index.js` 수정 후 Vercel 재배포 → CDN이 신규 파일 서빙 확인 (`utm_source` 6→5개 → 정상)
