---

## 개요

`growth-analytics`는 Google Analytics, 유저 히트맵 등의 성장 분석 기능을 구축하기 위한 신규 프로젝트입니다.
인증 시스템(로그인/회원가입/비밀번호 관리)을 `ads-library`의 구현체에서 복사하여 이식하였습니다.

## 인증 아키텍처

### 인증 흐름

```
사용자 접근
    ↓
App.js (AuthProvider 감쌈)
    ↓
AuthContext.js - Supabase 세션 감지
    ↓
세션 있음 → fetchUserMetadata() → users 테이블 조회 → 역할/조직 정보 로드
세션 없음 → /auth/sign-in 리디렉트
```

### 라우트 구조

```
/auth/sign-in          로그인
/auth/sign-up          회원가입 (초대 코드 또는 셀프)
/auth/forgot-password  비밀번호 찾기
/auth/reset-password   비밀번호 재설정 (이메일 링크 경유)
```

### AuthContext 제공 값

```javascript
// 상태
user              // Supabase auth 유저 객체
loading           // 초기 로딩 여부
role              // 'master' | 'agency_admin' | 'agency_manager' | 'advertiser_admin' | 'advertiser_staff' | 'viewer'
organizationId    // 소속 조직 ID
advertiserId      // 소속 광고주 ID
organizationType  // 'advertiser' | 'agency'
userName          // 사용자 이름
availableAdvertisers  // 접근 가능한 광고주 목록
currentAdvertiserId   // 현재 선택된 광고주 ID

// 메서드
signIn(email, password)
signUp(email, password, inviteCode)
signOut()
switchAdvertiser(advertiserId)

// 권한 체크 함수
isMaster()
canAccessSuperAdmin()
canAccessBrandAdmin()
canInvite()
isAgency()
isAdvertiser()
isAdvertiserAdmin()
```

## 회원가입 방식

### Phase 1 (현재): 초대 전용

`PHASE_1_INVITE_ONLY = true` 플래그로 셀프 회원가입 차단.
초대 코드(`invitation_codes` 테이블)를 통해서만 가입 가능.

**초대 코드 타입 (`invite_type` 필드)**

| 타입 | 설명 |
|------|------|
| `new_organization` | 신규 광고주 조직 + 브랜드 생성 |
| `new_brand` | 기존 조직에 신규 브랜드 추가 |
| `new_agency` | 신규 대행사 조직 생성 |
| (없음/기타) | 기존 조직에 사용자 추가 |

### Phase 2 (예정): 셀프 회원가입

`SelfSignUpForm.jsx`에 Supabase 연동 로직 주석으로 작성되어 있음.
`PHASE_1_INVITE_ONLY = false`로 변경 시 활성화.

## Supabase 설정

### 현재 상태

- Supabase 프로젝트 미생성 상태
- `.env`의 `REACT_APP_SUPABASE_URL`, `REACT_APP_SUPABASE_ANON_KEY` 비어 있음
- `src/config/supabase.js`에서 미설정 시 `console.warn`만 출력하고 앱은 정상 실행

### 향후 Supabase 연동 시 필요한 테이블

```sql
-- 필수 테이블
users               -- 사용자 정보 (id, email, name, role, organization_id, advertiser_id, status)
organizations       -- 조직 (id, name, type)
advertisers         -- 광고주/브랜드 (id, name, organization_id)
invitation_codes    -- 초대 코드 (code, invite_type, invited_email, role, expires_at, used_by)
```

### Edge Functions (비밀번호 재설정 이메일)

- `send-password-reset-email` — 비밀번호 찾기 페이지에서 호출

---

## 비밀번호 재설정 흐름

```
1. /auth/forgot-password 에서 이메일 입력
2. supabase.functions.invoke('send-password-reset-email') 호출
3. 사용자 이메일로 재설정 링크 발송
4. 링크 클릭 → /auth/reset-password 접속
5. Supabase 세션 자동 감지 (최대 15번 재시도, 2초 간격)
6. 새 비밀번호 입력 → supabase.auth.updateUser() 호출
7. 완료 후 3초 뒤 /auth/sign-in 리디렉트
```
